## 对比结论

本次人工审查与代码审查的结论方向总体一致，均认为该文件**不宜按现文本直接发布**，主要风险集中在：

- 评分标准设置不当
- 商务和履约责任明显偏重
- 现场踏勘和驻场要求可能形成隐性限制

但两者在**问题抽象质量、证据落点和噪声控制**上差异明显：

- 人工审查更接近采购人改稿场景，能把问题压成少数核心主问题。
- 代码审查已经能抓住部分评分和商务主线风险，但仍存在：
  - 人员画像类问题识别不足

  - 部分问题标题过宽或定位偏移

## 一致的部分

### 1. 评分标准存在明显高风险

人工审查认为：
- 技术项 63 分偏高
- 多项方案采用 `3 / 2.8 / 2.6` 式主观打分
- 保险赔付限额、信息化软件著作权、重大安全事故承诺等内容与核心物业履约能力关系较弱

代码审查对应命中：
- `F-002` 评分或资质条款中存在与标的域不匹配的证书、案例或模板内容
- `F-008` 至 `F-011` 评分中设置与履约弱相关的荣誉资质加分
- `F-012`、`F-013` 评分内容与评分主题或采购标的不完全匹配

判断：
- **代码审查已经具备较强的评分风险初筛能力**
- 但它更擅长抓“荣誉/证书/错位评分”，对“整体评分结构过主观、技术分过高”的表达还不如人工稳定

### 2. 商务和履约责任失衡问题方向一致

人工审查认为：
- 人员工资和社保承诺叠加无效投标、扣款、终止合同
- 大量运行成本、物资、活动保障和实验辅助性事务转嫁给中标人
- 验收和付款条款明显偏向采购人

代码审查对应命中：
- `F-003` 商务责任和违约后果设置明显偏重
- `F-005` 商务条款设置异常资金占用安排

判断：
- **代码审查已经能抓到“商务链条偏重”这条主线**
- 但它当前收得仍偏宽，对“工资社保承诺过重”“费用广泛转嫁”“验收解释偏向采购人”还没有拆成更贴近人工改稿的子问题

### 3. 现场踏勘 / 驻场要求存在隐性限制风险

人工审查认为：
- 文件将现场踏勘和方案评分结合
- 又对踏勘设置介绍信、身份证原件和报名网页打印件要求
- 容易形成事实上的隐性门槛

代码审查对应命中：
- `F-004` 驻场、短时响应或服务场地要求形成事实上的属地倾斜

判断：
- **方向一致**
- 但人工审查更准确地指出了“现场踏勘 + 方案评分”的组合风险；代码审查当前更偏向把它泛化成“属地倾斜”

## 代码审查当前的明显误判或噪声

### 1. `F-001` 定位明显不准

`F-001` 标题是：
- `资格条件设置经营年限、属地场所或单项业绩门槛`

但它引用的原文却是：
- `地址：福州市通湖路330号；地址：福州市鼓楼区华林路128号屏东写字楼17层`

这更像是招标邀请中的地址信息，而不是实质资格门槛。

判断：
- **这是一个明显的定位错误**
- 说明代码审查在资格章节附近，仍可能被结构识别或字段抽取误导

### 2. `F-006` 命中了电子投标文件格式，属于模板/格式污染

`F-006` 被上浮为：
- `资格条件整体超出法定准入和履约必需范围`

但证据位置在：
- `第七章 电子投标文件格式-二、投标人的资格及资信证明文件`

这更像是投标文件格式模板，而不是本项目真正新增的资格门槛。

判断：
- **这是典型的模板/格式污染**
- 说明当前 `tender_document_parser + requirement_scope_layer` 已经有帮助，但对格式章节的降权还不够彻底

### 3. `F-015`、`F-016` 也是格式章节噪声

这两条将：
- `投标标的为货物时填写品牌`

这类电子投标文件格式文本，上浮成了：
- `安全环保类技术要求可能合理但需补充必要性论证`

判断：
- **这是当前代码审查里最典型的伪风险**
- 也说明模板/格式文本与实质需求正文仍未完全切干净

### 4. `F-004` 方向对，但收得过宽

代码把：
- 三班制
- 24小时值班电话
- 驻场和服务场地要求

统合成：
- `驻场、短时响应或服务场地要求形成事实上的属地倾斜`

人工审查则更精确地把风险落在：
- `现场踏勘和方案评分组合`
- `24小时值班与驻场要求是否必要`

判断：
- **代码有方向感，但问题边界仍然偏泛**

## 总体判断

### 代码审查已经具备的能力

当前代码审查已经能稳定初筛出：
- 评分中荣誉、证书、信息化能力、承诺函等错位加分
- 商务条款整体偏重
- 驻场/响应类要求可能形成限制
- 环保和安全类要求需要必要性论证

这说明它已经具备了：
- **评分风险发现能力**
- **商务失衡发现能力**
- **文件初筛能力**

### 当前仍然明显落后于人工审查的能力

当前仍落后的关键点是：
- 人员画像限制识别
- 模板/格式污染压制
- 费用转嫁和品牌倾向拆分
- 验收解释条款单独识别
- 问题标题的人工化收束

## 一句话结论

这份文件上，代码审查已经能看出“评分”和“商务”两条主线风险，但距离人工审查还有明显差距：

- **人工审查更强在于看出人员画像限制、费用转嫁边界和验收解释失衡**
- **代码审查更像离线初筛器，能先指出风险方向，但还不足以直接替代人工形成最终审查意见**

当前最值得继续补的，不是再加更多宽规则，而是：
- 人员画像限制识别
- 模板/格式污染压制
- 商务费用与验收边界子主题拆分
