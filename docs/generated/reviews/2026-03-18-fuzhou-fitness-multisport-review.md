# 2025年省级全民健身工程（多功能运动场项目） 人工式审查意见

## 一、总体结论

总体结论：`高风险，不建议按现文本直接发标。`

本项目虽然以“服务类”竞争性磋商文件形式编制，但采购核心标的是多功能运动场整体器材及安装调试，属于明显的货物供货并安装调试场景。现有文本主要风险不在资格准入，而在评分结构失衡、技术参数与检测证明路径过严、售后响应时限偏紧、验收及费用边界明显偏向成交供应商承担。

## 二、主要问题

### 1. 评分结构明显失衡，技术评分过重且大量依赖负偏离扣分

- 位置：评审的标准和方法
- 页码提示：第10页左右
- 辅助行号：428-457
- 原文摘录：
  - `技术部分评分PT 满分为78.0000分`
  - `标注“▲”的技术要求（共6项）每负偏离一项扣3分，共计18分`
  - `未标注“★”、“▲”号的技术要求（共74项）每负偏离一项扣0.7分，共计51.8分`
- 风险类型：`excessive_scoring_weight`
- 风险等级：高
- 合规判断：`likely_non_compliant`
- 适用逻辑：报价仅 `10` 分，技术部分高达 `78` 分，且大量采用逐项负偏离扣分方式，本质上会把评审重心锁定在参数明细和证明材料齐备程度上，弱化价格竞争和整体履约能力比较。
- 修改建议：压降技术分值总量，减少细项逐条扣分式设计，将关键性能指标收束为少数核心项。

### 2. 技术评分中叠加大量专项检测要求，证明路径明显过严

- 位置：第三章 采购内容及要求-二、技术要求
- 页码提示：第20-23页左右
- 辅助行号：867-991
- 原文摘录：
  - `须提供具有CMA和CNAS标识的检测（检验）报告复印件作为佐证材料`
  - `并出具SGS检测报告`
  - `并提供全国认证认可信息公共服务平台网站的查询结果或截图`
- 风险类型：`technical_justification_needed`
- 风险等级：高
- 合规判断：`needs_human_review`
- 适用逻辑：对灯具、硅PU、环保指标等条款叠加 `CMA/CNAS`、`SGS`、平台截图等证明形式，会把合规证明进一步收窄为特定第三方检测路径，明显抬高举证成本。个别指标可能有必要，但当前文本没有给出充分必要性论证。
- 修改建议：将证明要求改为能证明性能达标的等效资料，不宜普遍要求特定检测机构标识、截图和专项报告叠加。

### 3. 技术要求中存在明显定向和品牌痕迹

- 位置：第三章 采购内容及要求-二、技术要求
- 页码提示：第22页左右
- 辅助行号：887-949
- 原文摘录：
  - `6、品牌：leyard`
  - `硅PU原料中有害物质限量须符合GB36246-2018`
  - `重点管控新污染物...不得检出`
- 风险类型：`narrow_technical_parameter`
- 风险等级：高
- 合规判断：`likely_non_compliant`
- 适用逻辑：在核心产品硅PU条款中直接出现具体品牌表述，同时叠加高强度环保和新污染物检测要求，容易形成特定品牌或少数供应商更易满足的效果。
- 修改建议：删除品牌指向性表述，仅保留经论证确有必要的性能指标，并采用中性描述。

### 4. 智能显示屏、OTA、扫码报修等功能叠加到健身器材中，边界外扩明显

- 位置：第三章 采购内容及要求-二、技术要求
- 页码提示：第23页左右
- 辅助行号：999-1016
- 原文摘录：
  - `每站器材均装置有智能显示屏`
  - `智能显示屏支持OTA升级`
  - `可通过手机扫描二维码实现一键报修，查看运动指导视频`
- 风险类型：`mixed_scope_boundary`
- 风险等级：中高
- 合规判断：`potentially_problematic`
- 适用逻辑：本项目主标的是多功能运动场器材，但条款把智能屏、OTA、扫码报修、语音播报等信息化能力作为器材配置一并纳入，容易将普通体育器材采购抬升为“器材+软件+运维”混合项目，却没有同步写清接口、维护和预算边界。
- 修改建议：区分基础器材性能要求与额外智能化配套要求；如确需智能功能，应单独说明必要性、兼容性和后续运维边界。

### 5. 商务评分中的售后服务方案存在较强主观性

- 位置：评审的标准和方法-商务部分
- 页码提示：第11页左右
- 辅助行号：479-486
- 原文摘录：
  - `由评委进行评议`
  - `阐述项目实施基础方案内容同时在确保方案实施的基础上增加有利于项目执行内容的得3分`
- 风险类型：`ambiguous_requirement`
- 风险等级：中
- 合规判断：`potentially_problematic`
- 适用逻辑：售后服务方案用语偏概括，缺少量化锚点，容易导致评分依赖评委主观判断。
- 修改建议：将售后方案评分改为响应时限、人员配置、备件保障、故障处理流程等可核验指标。

### 6. 售后响应时限偏紧，形成事实上的本地化或既有服务网络倾向

- 位置：第三章 采购内容及要求-三、商务条件-售后服务要求
- 页码提示：第26页左右
- 辅助行号：1114-1117
- 原文摘录：
  - `4小时内响应，8小时内委派专业技术人员到现场`
  - `若无法按时响应处理，采购人可另行安排人员维修处理，维修产生的一切费用由成交供应商承担`
- 风险类型：`geographic_restriction`
- 风险等级：高
- 合规判断：`potentially_problematic`
- 适用逻辑：本项目为健身工程器材项目，确有售后需求，但 `4小时响应+8小时到场+不能按时处理即由采购人另行安排且费用全由供应商承担` 的组合要求偏重，容易形成对本地服务网络更成熟供应商的事实优势。
- 修改建议：按故障等级细分响应机制，区分电话响应、远程支持和现场到场，不直接将极短到场时限作为通用要求。

### 7. 验收费用和相关履约成本整体转嫁给成交供应商

- 位置：第三章 采购内容及要求-三、商务条件-安装、调试及验收
- 页码提示：第25-26页左右
- 辅助行号：1096-1107
- 原文摘录：
  - `成交供应商需派有设备调试经验的技术人员对设备进行调试`
  - `验收所发生的一切费用均由成交供应商承担`
- 风险类型：`one_sided_commercial_term`
- 风险等级：高
- 合规判断：`likely_non_compliant`
- 适用逻辑：出厂检验、自检、安装地验收、调试测试和“一切费用由成交供应商承担”被放在一条链上，会显著加重成交供应商履约成本与责任，不利于公平竞争。
- 修改建议：明确哪些费用属于供应商正常交付义务，哪些检测或专项验收费应由采购人承担或另行约定，不宜笼统表述为“一切费用”。

### 8. 验收标准和最终验收边界偏向采购人使用要求

- 位置：第三章 采购内容及要求-三、商务条件-验收程序和方法
- 页码提示：第26页左右
- 辅助行号：1097-1105
- 原文摘录：
  - `验收结果应符合采购人使用要求`
- 风险类型：`unclear_acceptance_standard`
- 风险等级：中高
- 合规判断：`potentially_problematic`
- 适用逻辑：验收应围绕采购文件、合同和明确技术指标展开，若再附加“符合采购人使用要求”的开放性标准，容易造成最终验收边界不清。
- 修改建议：将验收标准限定为采购文件、合同约定和对应国家/行业标准，不宜再增加过于开放的主观使用要求。

## 三、建议的总体整改方向

- 将项目整体定位明确为“货物供货并安装调试”，避免继续沿用“服务类项目”叙事影响评分和商务设计。
- 压降技术评分占比，减少细项逐条扣分，删除或弱化过度依赖第三方检测报告的评分设计。
- 删除品牌指向和无充分论证的高强度检测证明路径，改为中性、可替代的证明方式。
- 对智能显示屏、OTA、扫码报修等智能化扩展功能单独论证必要性并明确边界。
- 将售后响应、验收、调试、费用承担、违约后果拆开约定，避免叠加式压重成交供应商责任。

## 四、需要重点人工复核的事项

- 智能化配置是否确属本项目预算和采购目标的必要组成部分。
- 高强度环保、新污染物和专项检测要求是否已有充分的项目必要性论证。
- 售后 `4小时响应、8小时到场` 是否与项目实际运维场景相匹配。
- “验收所发生的一切费用”中具体包括哪些费用，是否存在应由采购人承担的部分。
