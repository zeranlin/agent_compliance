# 政府采购需求管理办法

## 官方来源

- 文件名称：政府采购需求管理办法
- 发布机关：财政部
- 官方链接：[财政部原文](https://www.mof.gov.cn/gkml/caizhengwengao/wg2021/wg202005/202109/P020210917565943122846.pdf)

## 审查中最常用的规则点

### 1. 采购需求应当合法、合规、合理

适用理解：
- 采购需求应当符合项目特点和实际需要
- 不应超出采购标的所必需的范围
- 技术要求和商务要求应尽可能客观明确

常见适用场景：
- 平台能力要求过度外扩
- 技术参数或服务要求明显超出核心标的
- 验收标准和服务标准写得过于模糊

### 2. 采购实施计划中的评审因素应与采购需求对应

适用理解：
- 评审因素应当与采购需求中的技术、商务要求对应
- 量化指标应明确
- 客观但不可量化的指标通常不适合作为评分项

常见适用场景：
- 评分项与采购标的不直接相关
- 平台演示、现场陈述、综合比较分值过高
- 增值能力、荣誉、奖项被当作高分项

### 3. 履约验收方案应明确

适用理解：
- 应明确验收主体、时间、方式、程序、内容和标准
- 验收标准应覆盖采购需求中的客观、量化指标

常见适用场景：
- 只写“动态考核”“不达标取消资格”但没有标准
- 使用“先进水平”“用户满意”这类主观验收语言

## 可直接复用的引用提示

```text
依据《政府采购需求管理办法》，采购需求应当符合项目特点和实际需要，技术和商务要求应当客观明确。当前条款将核心采购标的之外的综合平台能力、增值功能或主观展示要求纳入刚性要求/高分项，存在超出项目实际需要并限制竞争的风险。
```

```text
依据《政府采购需求管理办法》，履约验收方案应当明确验收内容、标准和方法。当前条款仅作原则性表述，缺少可核验指标，后续执行和争议处理风险较高。
```

## 使用提醒

- 这是高频核心依据，几乎适用于所有采购需求合规审查。
- 具体条款引用时，仍应结合项目类型、采购方式和问题类型说明适用逻辑。
