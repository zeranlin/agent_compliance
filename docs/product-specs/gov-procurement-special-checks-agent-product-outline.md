# 政府采购四类专项检查智能体产品初稿

## 1. 产品名称

- **政府采购四类专项检查智能体**


## 2. 第一版目标

围绕政府采购文件中的四类专项检查事项，输出结构统一的专项检查结论、命中问题、证据位置和整改建议。

第一版不追求：

- 替代完整人工审查
- 自动覆盖全部采购场景的专项政策变化
- 直接形成正式处罚或裁决意见


## 3. 核心输入

- 采购文件/采购需求文本
- 四类专项检查规则口径
- 正负样例
- 人工专项检查基准


## 4. 核心输出

- 四类专项检查结论
- 专项问题列表
- 证据位置
- 风险说明
- 整改建议


## 5. 第一版建议章节

- 专项检查概述
- 类别一检查结论
- 类别二检查结论
- 类别三检查结论
- 类别四检查结论
- 需人工复核事项


## 6. 第一版评测关注点

- 是否已固化四类专项检查的输出结构
- 是否能输出统一的专项结论模板
- 是否能带出证据位置与整改建议
- 是否比空白骨架更接近人工专项检查记录
