# 代码审查能力产品介绍

## 1. 产品是什么

本项目的代码审查能力，是一套面向政府采购文件合规审查的本地执行系统。

它不是简单的“关键词扫描器”，也不是纯聊天式大模型问答，而是一套可以直接运行的审查产品能力，能够：
- 读取采购文件
- 自动识别资格、评分、技术、商务、验收等风险
- 输出结构化问题清单
- 定位到原文件位置
- 给出风险说明和改稿建议
- 持续把新问题沉淀成规则候选和 benchmark

## 2. 它解决什么问题

它主要解决 4 类问题：
- 长文件人工首轮筛查成本高
- 不同审查人输出口径不一致
- 同类问题反复出现，难以形成可复用能力
- 审查结果很难与原文、规则和后续优化闭环起来

## 3. 适合谁用

适用对象：
- 采购需求编制人员
- 采购代理机构复核人员
- 法务与合规审核人员
- 信息化、医疗、工程、后勤等业务条线的内控人员
- 用于建设政府采购合规辅助系统的技术团队

## 4. 当前能做什么

当前代码审查可以覆盖：
- 资格条件
- 评分标准
- 技术要求
- 商务条款
- 验收与检测
- 付款与违约
- 模板残留与义务外扩

当前重点可识别的问题包括：
- 一般财务和规模门槛
- 经营年限、属地场所、单项业绩门槛
- 错位行业资质和专门许可
- 品牌档次评分
- 错位认证、荣誉和跨领域证书
- 主观评分分档
- 演示机制门槛
- 事实上的属地倾斜
- 标准错位和证明形式过严
- 异常资金占用
- 验收费转嫁
- 责任失衡
- 验收边界不清
- 模板错贴和义务外扩

## 5. 它怎么工作

简化流程：

1. 上传 `docx/pdf/txt`
2. 系统提取正文并切分条款
3. 规则引擎先做高频问题初筛
4. 结构分析器把碎点归并成章节级主问题
5. 如启用本地模型，则补边界判断和全文辅助扫描
6. 仲裁层收束结果
7. 输出审查意见、结构化 findings 和原文定位

## 6. 主要交付结果

产品输出的主要结果包括：
- 审查问题清单
- 风险等级排序
- 原文定位
- 代表性证据
- 风险说明
- 改写建议
- Markdown 审查报告
- JSON findings

## 7. 当前页面能力

当前本地 Web 支持：
- 上传文件审查
- 启用/关闭本地模型
- 启用/关闭缓存
- 渲染 `docx` 原文
- 点击问题定位原文
- 按章节查看主问题
- 区分规则命中与模型新增
- 管理候选规则与 gate 状态

新版审查页 `review-next` 已重点优化为：
- 高风险优先
- 资格/评分/技术/商务分组
- 文档定位与问题高亮
- 代表性证据 + 风险说明 + 建议改写

## 8. 当前与人工审查的关系

当前代码审查已经能够覆盖大量高频问题，并逐步从“问题扫描”进化到“章节级主问题审查”。

与人工审查相比，当前优势在于：
- 可重复
- 可追溯
- 可批量运行
- 可积累规则和 benchmark
- 可直接接入系统

当前仍在继续逼近人工的方面包括：
- 边界判断
- 章节级总结表达
- 代表性证据选择
- 混合采购场景下的自然拆分

## 9. 使用价值

对业务和采购侧的价值：
- 降低首轮复核成本
- 提前暴露显性高风险条款
- 输出更适合改稿的主问题
- 提高审查一致性

对技术与平台侧的价值：
- 可本地部署
- 可与内部系统对接
- 可沉淀规则和 benchmark
- 可持续自动增强

## 10. 当前边界

当前产品还不是“完全替代人工终审”的系统。

仍建议在这些场景保留人工复核：
- 地方规则依赖强
- 医疗、工程、信息化等混合采购边界复杂
- 需要确认行业必要性和市场可得性的技术要求
- 法律依据时效性和适用范围需要最终确认

## 11. 一句话介绍

可以对外简洁描述为：

“这是一套面向政府采购文件合规审查的本地代码化审查系统，能够自动识别资格、评分、技术和商务风险，输出可定位、可改稿、可持续优化的结构化审查结果。”
