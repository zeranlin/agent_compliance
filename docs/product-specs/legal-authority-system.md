# 法规依据体系

## 目标

为政府采购需求合规性审查智能体建立一套可引用、可更新、可追溯的法规依据体系，使其在输出问题判断时，不只给出风险结论，还能说明判断依据来自哪一层规则。

## 使用原则

- 先判断问题类型，再匹配规则依据。
- 先引用高位阶规则，再补充部门规章、规范性文件和地方细则。
- 无法稳定支持的规则，不应强行引用。
- 对法域差异明显的问题，应标记为人工复核。

## 依据分层

### 第一层：法律

主要用于确定基本原则、法定责任边界和采购活动底层要求。

建议重点覆盖：
- 《中华人民共和国政府采购法》
- 《中华人民共和国政府采购法实施条例》
- 《中华人民共和国民法典》
- 《中华人民共和国中小企业促进法》
- 《中华人民共和国数据安全法》
- 《中华人民共和国个人信息保护法》

适用场景：
- 公平竞争、非歧视、公开透明
- 采购人和供应商的基本权利义务
- 合同责任、违约责任
- 数据安全和信息保护

### 第二层：财政部部门规章和管理办法

主要用于支撑采购需求、评审规则、采购方式、框架协议等高频审查事项。

建议重点覆盖：
- 《政府采购需求管理办法》
- 《政府采购货物和服务招标投标管理办法》
- 《政府采购框架协议采购方式管理暂行办法》
- 《政府购买服务管理办法》
- 《政府采购促进中小企业发展管理办法》
- 与质疑投诉、信息公告、评审专家管理相关规则

适用场景：
- 资格条件是否与履约能力相关
- 评审因素是否来自采购需求且可量化
- 是否存在不合理加分或重复加分
- 框架协议适用条件、第二阶段成交规则
- 政府购买服务项目是否适用当前方式

### 第三层：规范性文件、答复口径和官方政策解读

主要用于理解规则适用逻辑、常见错误和监管关注点。

建议重点覆盖：
- 财政部公开答复
- 中国政府采购网政策解读
- 财政部或政府采购监管部门公开发布的问答口径

适用场景：
- 条文过于原则，需要理解实务口径时
- 判断某类加分、限制、演示要求是否常见违规时
- 形成“高风险但需谨慎引用”的解释性判断时

### 第四层：地方财政部门和省级政府采购平台规则

主要用于处理法域差异和地方执行细则。

建议重点覆盖：
- 省级财政厅实施细则
- 省级政府采购网发布的监督检查、投诉处理、操作指引
- 地方性框架协议或政府购买服务实施口径

适用场景：
- 本省特殊操作要求
- 采购流程细节
- 地方监管机关的典型认定逻辑

### 第五层：典型案例和投诉处理结果

主要用于训练审查判断，不宜替代高位规则，但非常适合支撑实务风险识别。

建议重点覆盖：
- 中国政府采购网典型案例
- 财政部门投诉处理决定
- 地方监管平台公开案例

适用场景：
- 同类业绩、奖项荣誉、信用等级、属地限制等高频争议
- 评审标准量化不足
- 平台演示、现场陈述主观打分

## 问题类型与依据映射

### 资格条件问题

重点审查：
- 资格条件是否属于法定条件或履约必需条件
- 是否把奖项、荣誉、规模、属地、分支机构设置为门槛

优先引用：
- 政府采购法及实施条例
- 需求管理办法
- 招标投标管理办法

### 技术或服务要求问题

重点审查：
- 是否指向特定品牌、平台、产品、专有能力
- 是否超出采购标的本身所必需
- 是否不可量化、不可验证

优先引用：
- 需求管理办法
- 框架协议或政府购买服务相关规则
- 数据安全、信息安全相关法律法规

### 评分标准问题

重点审查：
- 是否与采购需求直接相关
- 是否量化
- 是否把资格条件转化为评分项
- 是否主观比较过强

优先引用：
- 招标投标管理办法
- 需求管理办法
- 典型投诉案例与监管解释

### 合同、验收和商务条款问题

重点审查：
- 责任是否失衡
- 验收标准是否明确可核验
- 付款条件是否清晰
- 是否排除法定责任

优先引用：
- 民法典
- 政府采购法及实施条例
- 需求管理办法

## 引用输出规范

每条审查意见尽量使用以下结构：

- `法律/政策依据`
- `适用逻辑`
- `引用层级`

示例：

```text
法律/政策依据：依据《政府采购需求管理办法》关于采购需求应当符合项目特点和实际需要、技术要求应当客观明确的要求。
适用逻辑：当前条款将代理记账服务扩大为必须具备融资、并购、培训等综合平台能力，超出核心采购标的的必要范围，存在不合理限制竞争风险。
引用层级：财政部部门规章
```

## 不确定情形处理

遇到以下情形时，不直接给出确定性法律结论：
- 地方规则和国家规则可能存在衔接差异
- 文件引用了本地试行办法，但当前未核实有效性
- 行业许可、安全、保密、兼容性可能具有特殊必要性

此时应输出：
- 暂定风险判断
- 可能适用的规则方向
- 需要进一步核实的法规或事实

## 维护要求

- 为每类高频问题建立“优先法条清单”。
- 法规库每月至少复核一次有效性。
- 新增案例时，要补充“该案例对应的规则依据映射”。
