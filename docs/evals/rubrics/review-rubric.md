# Review Rubric

## 目的

本 rubric 用于评估智能体输出是否对政府采购合规审查真正有用、准确且可执行。

## 评分维度

每次审查运行从五个维度评分，默认每项分值范围为 `0` 到 `2`。

### 1. 问题识别准确性

- `2`：能够识别预期中的关键风险，且没有编造明显缺乏依据的问题
- `1`：识别出部分关键风险，但遗漏或误判了至少一个重要问题
- `0`：遗漏重大风险，或输出中大多数 finding 缺乏依据

### 2. 分类质量

- `2`：问题类型、严重度和合规判断与案例预期基本一致
- `1`：部分正确，但严重度或判断存在明显偏差
- `0`：问题分类和严重度整体不可靠

### 3. 推理可追溯性

- `2`：能清楚把每条 finding 对应到原始条款，并解释其重要性
- `1`：解释基本合理，但追溯链条不完整
- `0`：解释泛泛而谈，或未锚定到原始条款

### 4. 改写建议可用性

- `2`：改写建议中性、可执行，并保留了合法业务需求
- `1`：改写方向有帮助，但仍不完整或过于抽象
- `0`：未提供可用改写，或改写引入了新的问题

### 5. 升级复核克制性

- `2`：能恰当地对不确定、法域敏感问题进行人工升级，同时不过度升级
- `1`：部分升级判断值得商榷
- `0`：要么过度自信，要么几乎把所有问题都升级处理

## 综合解释

- `9-10`：在该案例集上已接近可投入使用
- `7-8`：方向良好，但需要针对性优化
- `4-6`：尚不可靠
- `0-3`：需要较大幅度重构

## 失败标签

在评分时，应为较弱输出附加一个或多个失败标签。

建议标签：
- `false_negative`
- `false_positive`
- `severity_too_high`
- `severity_too_low`
- `unsupported_legal_basis`
- `rewrite_too_vague`
- `rewrite_changes_business_need`
- `insufficient_traceability`
- `missed_human_escalation`
- `over_escalated`
