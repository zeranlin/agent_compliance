## 当前阶段判断

当前离线引擎已经具备：
- 文档标准化
- 高频规则扫描
- 本地知识检索
- finding 聚合
- 本地网页审查入口
- 本地模型小范围接入
- 本地模型三类局部任务接入：
  - 模板错贴与标的域不匹配
  - 评分结构判断
  - 商务链路联合判断
- 规则候选自动生成
- benchmark gate 结果输出

当前距离人工式审查仍主要差在：
- 资格条件细拆
- 评分结构分析
- 技术参数边界判断
- 商务责任和付款联动识别
- 差异样本反向迭代
- 局部模型输出的进一步降噪与归并

当前调整后的工作原则：
- 先补“查点覆盖率”和“问题找全率”，暂不把分值精算作为主线
- 先把各一级、二级审查查点盘清，再按查点矩阵逐项补规则、结构分析和局部模型判断
- 每次增强都要能映射回具体查点，而不是只做零散表达优化
- 不再按单个项目逐个补差异，而是优先升级审查主链路：规则初筛、全文辅助扫描、局部 LLM 判断和 finding 仲裁协同工作

## 下一阶段主线

围绕“让代码审查更接近人工式审查，并且能够自动持续优化”这一目标，后续不再以零散补规则为主，而切换为以下主线：

- 先把审查主链路升级为：
  - 条款分类
  - 规则初筛
  - 结构分析
  - 章节级 LLM 审查辅助
  - finding 仲裁
- 再把自动优化闭环固定下来：
  - 差异样本沉淀
  - 规则候选生成
  - benchmark gate
  - 人工仅确认入库

## 阶段一：差异驱动的规则增强

目标：
- 先补齐人工稳定能识别、代码仍明显漏报的高收益规则。

任务：
- 增加资格条件细拆规则
- 增加评分高权重规则
- 增加中标后补证规则
- 增加商务责任失衡规则
- 增加付款与抽检联动规则

完成标志：
- 在窗帘采购样例中，新增命中：
  - 主管单位同意函
  - 股权结构
  - 连续多年审计
  - 国家级特色企业
  - 三体系高分
  - 同类业绩高分
  - 绝对免责
  - 终验款与抽检绑定

## 阶段二：评分与商务结构分析

目标：
- 让代码从“命中单句”升级为“理解条款组结构”。

任务：
- 为评分项补充类别识别和权重识别
- 为样品评分、方案评分、认证评分、业绩评分补充结构分析
- 为商务要求补充分组：售后、交货、验收、责任、付款

完成标志：
- 同一评分项中的混合问题能稳定拆分
- 商务条款中的责任失衡和验收边界不再混成一条泛化 finding

## 阶段三：局部模型参与

目标：
- 只在复杂边界问题上引入本地模型，提升接近人工判断的能力。

任务：
- 技术参数边界判断接入本地模型
- 评分项结构性失衡的局部推理接入本地模型
- 商务责任边界判断接入本地模型
- finding 拆分/合并建议接入本地模型

完成标志：
- 模型调用量仍然较低
- 复杂文件的技术参数和商务条款判断质量明显提升
- 模板错贴、评分结构、商务链路三类任务已能在真实文件中产出新增 findings

## 阶段四：benchmark 与持续迭代

目标：
- 形成“人工差异样本 -> 回归测试 -> 新版本收益”的长期机制。

任务：
- 建立差异样本目录和字段规范
- 把人工 vs 代码差异转成 benchmark
- 为漏报和误报增加标准标签
- 输出阶段性 gap report

完成标志：
- 每轮增强都能通过固定样例验证
- 能持续看到人工与代码差距缩小
- 每轮模型新增问题都能自动生成规则候选并给出 benchmark gate 结论

## P0 / P1 / P2 任务拆分

### P0

- `scoring_structure_analyzer`
- `commercial_chain_analyzer`
- `document_audit_llm`
- `finding_arbiter`
- `domain_match_engine`

当前进展：
- `scoring_structure_analyzer` 已启动落地，开始把评分碎点归并为章节级主问题，并已在民生诉求服务平台（二期）项目样本中验证出：
  - 多个方案评分项大量使用主观分档且缺少量化锚点
  - 现场演示分值过高且签到要求形成额外门槛
  - 商务评分将企业背景和一般财务能力直接转化为高分优势
- `commercial_chain_analyzer` 已启动落地，开始把付款、履约评价、整改和解除合同条款归并为商务链路主问题，并已在民生诉求服务平台（二期）项目样本中验证出：
  - 付款条件与履约评价结果深度绑定且评价标准开放
- `domain_match_engine` 已启动落地，开始结合项目主标的域识别资格错位资质、评分错位证书认证和模板残留义务外扩，并已在民生诉求服务平台（二期）项目样本中验证出：
  - 资格条件中存在与标的域不匹配的资质或登记要求
  - 评分项中存在与标的域不匹配的证书认证或模板内容
  - 文件中存在与标的域不匹配的模板残留或义务外扩

完成标志：
- 评分、演示、付款、履约评价类问题开始以章节级主问题输出，而不是大量碎点输出
- 错位证书、错位认证和明显项目域不匹配内容能稳定成点

## 风险提醒

- 若只增加规则、不做结构分析，会继续出现“看到了问题但拆不准”。
- 若过早扩大模型参与范围，会损害复审一致性和离线成本控制。
- 若没有 benchmark，后续增强很难证明是真进步还是只是改了表达。
