# 代码审查逼近人工审查执行计划

## 目标

围绕“让代码审查持续迭代接近人工审核效果”这一目标，建立从差异样本沉淀、规则增强、结构分析、局部模型参与到 benchmark 回归的闭环。

## 当前阶段判断

当前离线引擎已经具备：
- 文档标准化
- 高频规则扫描
- 本地知识检索
- finding 聚合
- 本地网页审查入口
- 本地模型小范围接入

当前距离人工式审查仍主要差在：
- 资格条件细拆
- 评分结构分析
- 技术参数边界判断
- 商务责任和付款联动识别
- 差异样本反向迭代

## 阶段一：差异驱动的规则增强

目标：
- 先补齐人工稳定能识别、代码仍明显漏报的高收益规则。

任务：
- 增加资格条件细拆规则
- 增加评分高权重规则
- 增加中标后补证规则
- 增加商务责任失衡规则
- 增加付款与抽检联动规则

完成标志：
- 在窗帘采购样例中，新增命中：
  - 主管单位同意函
  - 股权结构
  - 连续多年审计
  - 国家级特色企业
  - 三体系高分
  - 同类业绩高分
  - 绝对免责
  - 终验款与抽检绑定

## 阶段二：评分与商务结构分析

目标：
- 让代码从“命中单句”升级为“理解条款组结构”。

任务：
- 为评分项补充类别识别和权重识别
- 为样品评分、方案评分、认证评分、业绩评分补充结构分析
- 为商务要求补充分组：售后、交货、验收、责任、付款

完成标志：
- 同一评分项中的混合问题能稳定拆分
- 商务条款中的责任失衡和验收边界不再混成一条泛化 finding

## 阶段三：局部模型参与

目标：
- 只在复杂边界问题上引入本地模型，提升接近人工判断的能力。

任务：
- 技术参数边界判断接入本地模型
- 评分项结构性失衡的局部推理接入本地模型
- 商务责任边界判断接入本地模型
- finding 拆分/合并建议接入本地模型

完成标志：
- 模型调用量仍然较低
- 复杂文件的技术参数和商务条款判断质量明显提升

## 阶段四：benchmark 与持续迭代

目标：
- 形成“人工差异样本 -> 回归测试 -> 新版本收益”的长期机制。

任务：
- 建立差异样本目录和字段规范
- 把人工 vs 代码差异转成 benchmark
- 为漏报和误报增加标准标签
- 输出阶段性 gap report

完成标志：
- 每轮增强都能通过固定样例验证
- 能持续看到人工与代码差距缩小

## 当前最优先的下一步

1. 新增资格条件细拆规则包  
2. 新增评分高权重与中标后补证规则包  
3. 新增商务责任失衡与付款联动规则包  
4. 建立差异样本格式并把窗帘项目纳入 benchmark

## 风险提醒

- 若只增加规则、不做结构分析，会继续出现“看到了问题但拆不准”。
- 若过早扩大模型参与范围，会损害复审一致性和离线成本控制。
- 若没有 benchmark，后续增强很难证明是真进步还是只是改了表达。
