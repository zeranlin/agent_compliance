# 审查主链路升级方案

## 背景

最近一轮柴油发电机组项目对比暴露出一个结构性问题：当前代码审查链路虽然已经具备规则扫描、局部 LLM 判断、规则候选生成和 benchmark gate，但整体上仍偏向“候选问题增强器”，还不是一个完整的自动审查编排器。

典型表现是：
- 规则未生成候选时，本地模型也无法稳定补点
- 评分高分能识别一部分，但资格条件异常、评分内容错位、技术与商务边界问题仍大量漏报
- 表格区块切分不稳时，会把履约反馈表和评审程序中的文本误当成评分问题

因此下一阶段不应继续按单个项目逐条补差异，而应把主链路升级为“规则、结构分析、全文辅助扫描、finding 仲裁”协同工作的新编排层。

## 目标

把当前审查流程升级为：

`文档标准化 -> 条款分类 -> 规则初筛 -> 结构化分析 -> LLM 审查辅助 -> finding 仲裁 -> 规则候选生成 -> benchmark gate -> 人工仅确认入库`

这里的关键变化是：
- LLM 不再只做局部润色或零散补点
- 新增“全文辅助扫描”，但只生成候选，不直接定案
- rules 与 LLM 的输出进入统一仲裁层，而不是谁先命中谁说了算

## 一、主链路分层

### 1. 文档标准化层

保持现有能力：
- `docx/pdf/txt` 转稳定文本
- 段落、表格、章节、行号、页码映射
- `section_path`、`table_or_item_label`、`page_hint`

### 2. 条款分类层

新增明确目标：
- 把每个条款先识别为 `资格条件 / 评分标准 / 技术要求 / 商务条款 / 验收 / 付款 / 违约 / 其他`
- 后续规则和 LLM 都按条款类型走不同策略

### 3. 规则初筛层

规则继续承担：
- 高频显性问题识别
- 候选条款召回
- 低成本稳定覆盖

当前新增重点：
- `qualification_anomaly_engine`
  - 行业错位资质
  - 一般财务门槛
  - 经营年限门槛
  - 单项合同金额门槛
- `scoring_content_mismatch_engine`
  - 方案评分混入案例或证明形式
  - 商务评分混入企业规模、资产、纳税额、成立年限
  - 评分项中出现与主标的不相称证书

### 4. 结构化分析层

不再只看单句，而要先组织上下文：
- 资格条件整段
- 评分表整张
- 技术要求主题块
- 商务链路主题块

结构化分析要解决：
- 某条到底是评分项还是履约反馈表
- 某个高分项到底是在评方案、评业绩还是评企业规模
- 某个商务条款是否与付款、验收、责任联动

### 5. LLM 审查辅助层

LLM 分成两种任务：

#### 局部任务
- 模板错贴与标的域不匹配
- 评分结构判断
- 商务链路联合判断

#### 全文辅助扫描
- 对整份标准化文档再做一轮候选问题扫描
- 只输出：
  - `candidate_findings`
  - `suspicious_sections`
  - `possible_domain_mismatch`
  - `possible_missing_rules`
- 不直接写入最终 findings

### 6. finding 仲裁层

新增统一仲裁规则：
- 规则先命中的问题，LLM 只能补解释或补拆分，不直接推翻
- LLM 新增问题必须保留：
  - 原文条款
  - 任务来源
  - 置信度
  - 候选 issue_type
- 同一条款或相邻条款的重复问题统一归并
- 非评分区块中的 `优/良/中/差` 等高噪词进入误报抑制

## 二、为什么不把“全文自由审”当主引擎

本项目需要 LLM 全文参与，但不适合让全文自由审直接输出最终结论，原因有三：

1. 一致性差  
同一输入在不同轮次可能波动较大。

2. 可复核性差  
很难解释某条 finding 是规则得出还是自由推理得出。

3. 成本控制差  
每次复审都重跑全文自由推理，会削弱缓存和增量复审价值。

因此推荐做法是：
- `全文辅助扫描` 用于找候选
- `局部任务` 用于做边界判断
- `仲裁层` 用于决定最终 finding

## 三、P0 实施范围

本轮优先做三件事：

### 1. 资格异常识别引擎

目标：
- 先把人工一眼可见的资格条件异常稳定抓住

覆盖范围：
- 行业错位资质和登记证书
- 一般纳税额门槛
- 一般经营年限门槛
- 单项合同金额门槛

### 2. 评分内容错位识别

目标：
- 不只看“高分”，还看“评的内容对不对”

覆盖范围：
- 方案评分混入工程案例
- 方案评分混入检测报告形式
- 商务评分混入人员规模、资产、纳税额、成立年限
- 评分项中出现明显跨领域证书和资质

### 3. 全文辅助扫描

目标：
- 在规则候选不足时，仍能把显性问题送进候选池

原则：
- 只补候选，不直接定案
- 重点补资格异常、评分内容错位、技术固定年份、人员来源限定、付款责任边界

## 四、后续阶段

### P1
- 标的域/条款域匹配引擎
- 技术主题 LLM 任务
- 商务主题 LLM 任务
- 误报抑制层增强

### P2
- 行业包
  - 柴油发电机
  - 医疗设备
  - 医用布草/窗帘
  - 信息化

## 五、成功标志

当以下现象出现时，说明主链路升级方向是对的：

- 代码不再只抓评分高分，而开始稳定识别资格异常和内容错位
- 全文辅助扫描能把原来“规则没命中、模型也补不到”的显性问题送进候选池
- 同一份文件中的误报显著减少，尤其是不再误抓履约反馈表和评审程序文本
- 模型新增问题能稳定进入规则候选和 benchmark gate，而不是只停留在一次性输出中
