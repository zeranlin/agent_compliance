# 审查一致性与缓存复用设计

## 目标

在政府采购需求合规性审查中，实现以下目标：
- 同一文件在相同条件下复审结果保持一致
- 尽量减少对大模型的重复全量推理
- 仅在必要场景调用大模型处理边界判断和改写
- 使复审过程可复现、可解释、可增量更新

## 总体原则

审查一致性不应主要依赖“再次调用大模型并希望其重复同样结论”，而应优先依赖以下机制：
- 标准化输入
- 规则版本固定
- 引用快照固定
- 结构化 findings 缓存
- 增量重算

## 四层执行结构

### 1. 文档标准化层

先把原始 Word、PDF、扫描件等文件转换为稳定的中间表示。

建议至少产出：
- `file_hash`
- `document_name`
- `normalized_text`
- `section_map`
- `clause_map`
- `page_map`
- `line_map`

这一层应尽量使用确定性方法完成，如文本提取、章节切分、编号识别、页码定位等，不优先依赖大模型。

### 2. 规则命中层

对标准化后的文本优先使用规则、模式和关键词进行初筛，识别高频问题。

典型规则包括：
- 指定品牌、型号、平台、专有能力
- 属地限制、本地网点、本地备案
- 奖项荣誉、信用等级、认证资质加分
- 性别、年龄、身高、外貌等人员画像限制
- 同类业绩范围越界
- 主观评分过重
- 验收标准缺失
- 绝对免责或责任失衡

这一层应输出：
- `rule_hit_id`
- `matched_clause_id`
- `issue_type_candidate`
- `matched_text`
- `rule_set_version`

### 3. 大模型判断层

仅在以下情形调用大模型：
- 规则命中后需要进行法律语义判断
- 条款存在上下文依赖，需做边界识别
- 需要生成修改建议
- 需要生成建议替代表述
- 需要整合多个条款形成一个 finding

不应让大模型每次从整份长文档重新开始自由推理。

### 4. 结果固化层

每次审查完成后，应保存结构化 findings，以便复审直接复用。

建议至少保存：
- `file_hash`
- `review_scope`
- `rule_set_version`
- `reference_snapshot_id`
- `output_template_version`
- `findings`
- `overall_risk_summary`

## 缓存策略

### 一、文档解析缓存

缓存内容：
- 标准化文本
- 章节定位
- 条款定位
- 页码与辅助行号

复用条件：
- 文件内容未变化
- 解析方式未变化

### 二、规则命中缓存

缓存内容：
- 条款命中的规则
- 规则命中的证据片段
- 命中时间和规则版本

复用条件：
- 文件标准化结果未变化
- 规则库版本未变化

### 三、finding 缓存

缓存内容：
- 问题标题
- 风险位置
- 风险类型
- 风险等级
- 合规判断
- 依据
- 修改建议
- 建议替代表述

复用条件：
- `file_hash` 未变化
- `review_scope` 未变化
- `rule_set_version` 未变化
- `reference_snapshot_id` 未变化
- `output_template_version` 未变化

若上述条件均满足，应优先直接复用 findings，不重复进行全量大模型推理。

## 增量复审机制

如仅部分条件发生变化，应仅重算受影响部分。

### 仅输出格式变化

处理方式：
- 不重跑规则命中
- 不重跑合规判断
- 直接基于已有 findings 重排输出格式

### 仅引用资料更新

处理方式：
- 只重算受影响的 issue type 或 finding
- 其他 findings 继续复用

### 仅文件局部变化

处理方式：
- 只对受影响条款重新切分、重新命中规则、重新调用模型
- 未变化条款沿用原结果

## 一致性判断标准

当以下条件均未变化时：
- 审查文件未变化
- 审查范围未变化
- 规则版本未变化
- 引用快照未变化
- 输出模板版本未变化

则复审结果应保持一致，至少包括：
- 主要问题标题一致
- 风险位置一致
- 原文摘录一致
- 风险类型一致
- 风险等级一致
- 合规判断一致
- 修改建议一致或仅作非实质性文字优化

## 变化说明机制

如果复审结果与历史结果不一致，必须说明变化来源，限定为以下几类：
- 输入文件变化
- 审查范围变化
- 规则版本变化
- 引用快照变化
- 输出模板变化

不得在上述条件均未变化的情况下，随意改变结论。

## 对后续执行的要求

后续审查应优先遵循以下顺序：
1. 先判断是否存在可复用的标准化输入
2. 再判断是否存在可复用的规则命中结果
3. 再判断是否存在可复用的 findings
4. 仅对未命中缓存或受变化影响的部分调用大模型
5. 输出结果后更新缓存和版本留痕

## 一句话总结

一致性不是靠“记住上次怎么说”，而是靠：
- 固定输入
- 固定规则
- 固定引用
- 固定模板
- 缓存 findings
- 增量复审

这样才能在保证结果稳定的同时，减少重复调用大模型推理。
