# 持续更新机制

## 目标

为政府采购需求合规性审查智能体建立一个可周期运行的更新机制，使其能持续吸收新法规、新答复口径、新典型案例，并把这些变化转化为审查能力。

## 更新对象

更新机制至少覆盖以下四类内容：

- 法律法规和部门规章
- 财政部门政策解读和公开答复
- 中国政府采购网及省级政府采购网的典型案例
- 内部审查失败样本和人工修正记录

## 核心信息源

### 国家级来源

- 财政部官网
- 中国政府采购网
- 中国政府采购报相关实务栏目

### 地方级来源

- 省级政府采购网
- 省财政厅官网
- 本地投诉处理、监督检查和政策通知栏目

### 内部来源

- 人工审查记录
- 历次误报、漏报清单
- 已改写并采用的合规版本条款

## 更新频率建议

### 每周任务

- 抓取新增典型案例
- 抓取新增政策解读或监管提醒
- 识别是否出现新的高频争议点

### 每月任务

- 复核法规有效性
- 汇总高频案例专题
- 更新规则映射表
- 更新 benchmark 案例集

### 每季度任务

- 对智能体近一阶段审查结果做误报/漏报分析
- 调整审查提示词和问题分类体系
- 输出一版“能力变化报告”

## 单次更新流程

### 1. 抓取

从预设网站抓取新增页面或人工导入链接。

### 2. 识别

判断新增内容属于：
- 新法规
- 规则解释
- 典型案例
- 地方口径
- 失败复盘样本

### 3. 结构化

提取核心字段，如：
- 标题
- 来源
- 发布时间
- 问题类型
- 条款原文
- 认定理由
- 法规依据
- 可复用审查规则

### 4. 入库

根据内容类型写入对应库：
- 法规库
- 案例库
- 失败样本库

### 5. 生成派生产物

每次更新后应自动生成或补充：
- 高风险问题清单
- 新增审查规则
- 新 benchmark 样例
- 待人工复核事项

### 6. 反馈到审查系统

将新增内容同步到：
- 审查规则说明
- 法规依据映射
- 评测集
- 改写示例集

## 建议输出产物

每轮更新至少形成以下结果之一：

- `月度规则更新摘要`
- `新增典型案例清单`
- `高频争议专题`
- `误报漏报复盘报告`

## 自动化建议

如果后续做自动化任务，建议把自动更新拆成三个独立任务：

### 任务一：法规与政策更新

职责：
- 扫描财政部和省财政厅网站
- 识别规则新增、失效、替换
- 输出法规变化摘要

### 任务二：案例更新

职责：
- 扫描中国政府采购网和省级平台
- 识别新增典型案例和投诉处理案例
- 输出结构化案例候选

### 任务三：能力校验

职责：
- 从新增案例中抽取测试题
- 运行现有审查规则做对比
- 形成误报、漏报和薄弱点报告

## 更新后的质量门槛

新增内容进入正式规则库前，至少应通过以下检查：

- 来源是否权威
- 规则是否仍有效
- 案例结论是否足够明确
- 是否能够提炼为通用审查规则
- 是否需要法域标记

## 人工介入点

以下情形必须保留人工确认：

- 规则冲突
- 地方特殊口径与国家规则关系不明
- 案例来源不稳定或表述模糊
- 智能体对同一问题给出互相矛盾结论

## 成功标准

持续更新机制建立后，智能体应逐步具备这些能力：

- 能说明“为什么这是问题”
- 能引用较稳定的法规或政策依据
- 能从新案例中学会新的风险模式
- 能把过去的审查错误转化为未来的评测样例
