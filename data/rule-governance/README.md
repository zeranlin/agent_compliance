# 规则治理层数据

本目录存放代码审查规则治理层的结构化配置。

当前包括：

- `rule-priority-profile.json`
  - 定义默认规则家族优先级
  - 定义按审查领域的规则家族、问题类型和具体规则加权
  - 定义按审查领域的显式停用和降权策略
  - 用于 `catalog_sensitive_rule_router`

## 当前目标

这一层不是替代 `rules/*.py`，而是为现有规则体系补一层：

- `rule_registry`
- `rule_priority_profile`
- `catalog_sensitive_rule_router`

使规则具备：

- 可治理
- 可分级
- 可按采购品目和审查领域路由

## 当前策略

当前已解决：

- 不同采购品目下规则家族优先级不同
- 同一条款中同一风险簇有多条规则可命中时，按品目优先级只保留最合适的一条
- 正式规则状态分层
- 按品目显式停用或降权
- 规则管理页展示正式规则治理摘要

后续会继续补：

- 规则位阶
- 规则适用品目范围
- 实验规则和正式规则治理
- 规则收益与副作用评估
